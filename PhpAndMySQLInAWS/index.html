Какво е AWS

Amazon Web Services (AWS) е платформа, която предоставя отдалечени компютърни услуги на индивидуални клиенти, компании и правителства, предлагана от Amazon от 2006 година.

Едно от ключовите предимства на cloud computing е възможността да заменят първоначалните разходите за инфраструктура с ниски променящи се разходи, спрямо мащаба на бизнеса. С Cloud, компаниите вече няма нужда да планират и да се снабдяват със сървъри и други IT инфраструктури, седмици и дори месеци предварително. Вместо това, те могат моментално да вдигнат стотици, и дори хиляди сервъри за минути, и да постигнат по-бързи резултати.

Предимства на AWS

Днес, AWS осигурява високо надеждна, мащабируема, евтина инфраструктурна платформа в облака, която захранва стотици хиляди фирми в 190 страни по света. С центрове за данни в САЩ, Европа, Бразилия, Сингапур, Япония и Австралия, клиентите от всички индустрии се възползват от следните предимства:

Фиг. 2 Амазон Уеб Услуги - Предимства
  • Ниски цени

    AWS предлага ниско, pay-as-you-go ценообразуване, без предплащане или дългосрочно обвързване.
    Amazon успява да изгради и управлява глобална инфраструктура 'at scale' и да прехвърли спестяване
    на разходи под формата на ниски цени.

  • Гъвкавост и незабавна еластичност

    AWS осигурява глобална облачна инфраструктура, която позволява за бързо обновяване, експериментиране
    и итерейтване. Вместо да чакате седмици, дори месеци за хардуер, веднага можете да създадете (deploy)
    нови приложения, незабавно 'scale up' докато натоварването ви нараства, и незабавно 'scale down' набаза
    търсенето. Независимо дали имате нужда от една виртуална машина или от хиляди, независимо дали ви
    трябват за няколко часа или 24/7, плащате само за това което използвате.

  • Свобода

    AWS позволява вие да изберете платформата за разработка или моделът за програмиране, най-подходящ
    за вашия бизнес. Вие избирате колко услуги да използвате и как да ги използвате. Тази гъвкавост ви
    дава възможност да се фокусирате върху иновациите, а не инфраструктурата.

  • Сигурност

    AWS е сигурна, трайна технологична платформа с признати в индустрията сертификати и одити:
    PCI DSS Level 1, ISO 27001, FISMA Moderate, FedRAMP, HIPAA, and SOC 1 (преди позната като SAS 70 и/или
    SSAE 16) and SOC 2 одитни доклади. Услугите и дата центровете имат многослойна оперативна и физическа
    сигурност, за да гарантира целостта и безопасността на вашите данни.

Глобална Инфраструктура

AWS Cloud обхваща 61 Availability Zones (AZs), разпределени в 20 географски зони по цели свят, с обявени планове за още 15 AZs и 5 региона в Бахрейн, Кейптаун, САР Хонконг, Джакарта и Милано.

PHP в AWS

В AWS може да намерите всички необходими инструменти, документация и примерен код, за да разработвате сигурни, надеждни и мащабируеми PHP приложения в облака.

Какво е AWS SDK

AWS SDK за PHP улеснява достъпа на програмистите до уеб услугите на Амазон през техния PHP код. Така те могат да създават стабилни приложения и софтуер като използват услуги като Amazon Simple Storage Service (S3), Amazon Elastic Compute Cloud (EC2), Amazon DynamoDB, Amazon Glacier, и т.н.

Конфигуриране на AWS SDK за PHP

Настройки

Започнете с регистрация в AWS, вземете вашите AWS credentials и настройте стредата си.

За да се регистрирате за продукт, изпълнете стъпките:

  1. Отидете на началната страница на продукта, който желаете, например: aws.amazon.com/s3/.
  2. Натистене бутона sign-up в горния десен ъгъл на страницата.
  3. Следвайте инструкциите.

За да можете да използвате AWS SDK за PHP, трябва да имате AWS Access Key ID и Secret Access Key.
Стъпки:

  1. Отидете на: aws.amazon.com.
  2. Натистене Account и след това Security Credentials
  3. Намерете Access Credentials и изберете таба Access Keys
    AWS Access Key ID се появява в колоната Access Key
  4. За да видите Secret Access Key, натиснете Show

Важно! Това са тайни ключове, които само вие и AWS трябва да знаят. Много е важно да ги държите конфиденциално, за да защитите акаунта си.

Минимални изисквания за настройване на средата:

  1. PHP 5.2 или по-нова версия (силно препоръчително версия поне 5.2.12 или по-висока).
    Включивелно SimpleXML, JSON, PCRE и SPL.
  2. cURL PHP разширение (Компилирани с OpenSSL библиотеки за поддръжка на HTTPS.
  3. Възможност за четене от, и писане върху файловата система чрез file_get_contents() and file_put_contents().

Ако нямате работеща PHP среда, може да следвате стъпките от този линк и много лесно и безплатно да си вдигнете инстанция с конфигурирани PHP и MySQL.

Ако не сте сигурни за всички PHP инсталации, можете да проверите с AWS SDK for PHP Compatibility Test.

Инсталация на SDK

Има няколко начина за инсталация на AWS SDK за PHP

  • Използвайте инсталационния файл от aws.amazon.com/sdk-for-php/
  • Инсталация през Git
    • git clone https://github.com/aws/aws-sdk-php.git AWSSDKforPHP
    cd ./AWSSDKforPHP
  • Инсталация през Subversion
    • svn co http://svn.github.com/amazonwebservices/aws-sdk-for-php.git AWSSDKforPHP
    cd ./AWSSDKforPHP
  • Инсталация през PEAR
    • sudo pear channel-discover pear.amazonwebservices.com
    sudo pear install aws/sdk
Конфигуриране на SDK Security Credentials

Има два начина за Конфигуриране на вашите ключове, като единият е по-лесен, а другият - по-гъвкав. Почти винаги за предпочитане е по-простият начин, освен когато използвате PEAR. Ако изберете PEAR и съответно по-гъвкавия начин, можете да намерите копие на config-sample.inc.php в GitHub.

За да добавите вашите AWS credentials по лесния начин
  1. Намерете вашият config-sample.inc.php файл в SDK директорията и го преименувайте на config.inc.php
  2. Отворете файла с текстов редактор по ваш избор и там ще намерите описание на всички конфигурационни стойности.
  3. Когато SDK е зареден, първо ще потърси за config.inc.php файл във същата директория като sdk.class.php.
    Вече сте готови да стартирате примера!
За да добавите вашите AWS credentials по гъвкавия начин (PEAR)
  1. Създайте нов конфигурацинен файл в~/.aws/sdk/config.inc.php
    Можете или да свалите и копирате съдържанието от файла config-sample.inc.php, или да го копирате директно от горния GitHub линк.
  2. Отворете файла config.inc.php с текстов редактор по ваш избор. Там ще намерите описание на всички конфигурационни стойности.
  3. Напишете echo getenv('HOME');, за да се уверите, че PHP има достъп до HOME environment variable.
  4. Ако предходната команда не върне правилния път до файла, конфигурирайте го с: putenv('HOME=');.

RDS

Amazon Relational Database Service (Amazon RDS) улеснява настройването, работата и мащабирането на релационна база данни в облака. Осигурява икономична, с променящ се капацитет, като същевременно автоматизира административни задачи, отнемащи време, като например осигуряване на хардуер, настройки на базата данни, корекции и архивиране.

Amazon RDS е налична в няколко вида бази данни - оптимизирана за памет, производителност или входно/изходни операции - и ви дава възможност да избирате от шест познати системи за управление на бази от данни, включващи:

Можете да използвате AWS Database Migration Service, за да мигрирате или репликирате съществуващите си бази данни в Amazon RDS.

MySQL

MySQL е най-популярната в света релационна база данни с отворен код и Amazon RDS улеснява работата, настройването и мащабирането като позволява внедряване в облака. С Amazon RDS можете да пуснете мащабируеми MySQL сървъри само за минути, с рентабилен и лесно променящ се хардуер.

Amazon RDS за MySQL позволява да се фокусирате върху разработването на приложения, като управлява времеемките административни задачи свързани с базата данни, вкючително резервни копия, софтуерни корекции, мониторинг, мащабиране и репликация.

Предимства
Лесно, управлявано внедряване

Отнема само няколко клика в конзолата за управление на AWS, за да стартирате и да се свържете към готова за производство MySQL база данни само за минути. Amazon RDS за MySQL базите данни са с предварително конфигурирани параметри и настройки за избрания от вас тип сървър. Групираните параметри на базата данни осигуряват гранулярен контрол и фина настройка на вашата MySQL база данни.

Бързо, предвидимо съхранение

Amazon RDS предоставя две опции за съхранение, поддържани от SSD, за вашата MySQL база данни. General Purpose storage осигурява икономично съхранение за малки и средно-големи натоварвания. За високопроизводителни OLTP приложения, Provisioned IOPS предоставя постоянна производителност до 40,000 входно/изходни операции в секудна. С нарастването на изискванията ви за съхранение, можете да осигурите допълнително памет в движение с нулево закъснение.

Резервни копия и възстановяване

С функцията за автоматизирано резервно копие на Amazon RDS, можете да възстановявате вашата MySQL база данни до всеки един момент от зададения период на съхранение до 35 дни. Освен това можете да извършвате инициирани от вас резервни копия. Тези пълни резервни копия на вашата база данни ще се съхраняват от Amazon RDS, докато не ги изтриете лично.
